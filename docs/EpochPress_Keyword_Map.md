## Implementation Notes

- Epoch Press uses a **different codebase** from the TCM sites — this is NOT the multi-site Supabase system
- The `[slug]` dynamic route and `site_seo_pages` table pattern from the TCM system should be **adapted** for this codebase
- Two core landing pages are needed because Epoch Press has two physical locations
- Service pages should NOT be city-specific — they serve national/regional customers, not local walk-ins
- The `/products` page already exists — service SEO pages complement it with keyword-targeted content
